简介

代码构成

  1. .json 后缀的 JSON 配置文件

    1. JSON 是数据格式,不是编程语言
    2. 静态配置
    3. 内容:
      1. app.json:全局配置:所有页面路径、界面表现、网络超时时间、底部 tab–字段:pages 页面路径–window 字段:顶部背景颜色，文字颜色定义等
      2. project.config.json:工具配置;个性化配置(换电脑
      3. page.json:页面配置;独立定义每个页面的一些属性
    4. 语法:JSON 文件都是被包裹在一个大括号中 {}，通过 key-value 的方式来表达数据。JSON 的 Key 必须包裹在一个双引号中，在实践中，编写 JSON 的时候，忘了给 Key 值加双引号或者是把双引号写成单引号是常见错误。

  2. .wxml 后缀的 WXML 模板文件:类似 html

    1. 数据绑定: {{}};大小写敏感;属性值的绑定;未定义/undefined 不会同步到 wxml

    2. 可在{{}}进行简单的逻辑运算;三元;算数;字符串拼接

    3. 条件逻辑:\ True \ \ 3 \

    4. 列表渲染:

    5. 模板:

    6. 引用

      WXML 提供两种文件引用方式 import 和 include

    7. 共同属性

      | id | String | 组件的唯一标识 | 整个页面唯一 |
      | ———— | ———— | ————– | —————————————- |
      | class | String | 组件的样式类 | 在对应的 WXSS 中定义的样式类 |
      | style | String | 组件的内联样式 | 可以动态设置的内联样式 |
      | hidden | Boolean | 组件是否显示 | 所有组件默认显示 |
      | data-* | Any | 自定义属性 | 组件上触发的事件时，会发送给事件处理函数 |
      | bind/catch | EventHandler | 组件的事件 | |

  3. .wxss 后缀的 WXSS 样式文件:WXSS 具有 CSS 大部分的特性，小程序在 WXSS 也做了一些扩充和修改。rpx 全局样式和局部样式 支持部分选择器

    1. 文件组成:

      1. app.wxss(项目公共样式);page=>index.wxss(页面样式);其他

      2. 尺寸单位:rpx 小程序编译后，rpx 会做一次 px 换算。换算是以 375 个物理像素为基准，也就是在一个宽度为 375 物理像素的屏幕下，1rpx = 1px。

        举个例子：iPhone6 屏幕宽度为 375px，共 750 个物理像素，那么 1rpx = 375 / 750 px = 0.5px。

      3. wxss 引用 @import ‘./test_0.wxss’

      4. 内联样式:

      5. 选择器

  1. .js 后缀的 JS 脚本逻辑文件

    1. 小程序 js 构成:ecmascript;小程序框架;小程序 api

    2. 执行环境 ios/安卓/小程序 IDE

    3. 模块化:

    4. 脚本执行顺序

      1. 小程序的执行的入口文件是 app.js 。并且会根据其中 require 的模块顺序决定文件的运行顺序
      2. 当 app.js 执行结束后，小程序会按照开发者在 app.json 中定义的 pages 的顺序，逐一执行

    5. 作用域

      1. 在文件中声明的变量和函数只在该文件中有效，不同的文件中可以声明相同名字的变量和函数，不会互相影响
      2. 当需要使用全局变量的时，通过使用全局函数 getApp() 获取全局的实例，并设置相关属性值，来达到设置全局变量的目的

宿主环境

渲染层和逻辑层

定义:微信客户端给小程序所提供的环境

小程序的运行环境:渲染层(wxml/wxss)和逻辑层(js 脚本);

渲染层:使用了 WebView 进行渲染,多个页面=>存在多个 WebView 线程

逻辑层:JsCore 线程运行 JS 脚本

这两个线程的通信会经由微信客户端（下文中也会采用 Native 来代指微信客户端）做中转,

逻辑层发送网络请求也经由 Native 转发，小程序的通信模型下图所示。

程序与页面

整个小程序只有一个 App 实例，是全部页面共享的

微信客户端会先根据 logs.json 配置生成一个界面，顶部的颜色和文字你都可以在这个 json 文件里边定义好。紧接着客户端就会装载这个页面的 WXML 结构和 WXSS 样式。最后客户端会装载 logs.js，你可以看到 logs.js 的大体内容就是:

Page 是一个页面构造器，这个构造器就生成了一个页面。在生成页面的时候，小程序框架会把 data 数据和 index.wxml 一起渲染出最终的结构，于是就得到了你看到的小程序的样子。

在渲染完界面之后，页面实例就会收到一个 onLoad 的回调，你可以在这个回调处理你的逻辑
